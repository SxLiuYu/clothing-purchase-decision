## 一、产品定位与超级能力

### 1.1 产品定位（升级版）

**一句话定义：**
> 你的 7×24 穿搭决策代理——不是“推荐一套衣服”，而是计算“你此刻最应该穿什么，以及这件新衣服值不值得买”。

| 维度 | 说明 |
|------|------|
| 产品名称 | 智搭配 / StyleAgent（建议） |
| 目标用户 | 22-40 岁，关注穿搭但时间有限、决策成本高的都市人群 |
| 核心痛点 | 满柜子衣服仍觉得“没衣服穿”；冲动买新衣回家发现无法搭配；不知道旧衣服为什么该留该扔 |
| 核心价值 | 穿衣决策代理（穿什么/买不买/衣柜优化） |
| 差异化定位 | 身材优先 + 组合缺口计算 + 动态体态档案 + 穿搭决策可解释 |
| 商业模式 | CPS 分佣（主要）+ 高级订阅 + 品牌合作 + 穿搭师服务 |

### 1.2 三大超级能力（Superpowers）

#### Superpower 1：穿搭决策中枢（Style Decision Copilot）
- 不是：返回3套好看的穿搭图片。
- 而是：基于天气、行程、体型、衣橱存量、穿着历史，推演一条“可执行穿搭决策链”。
- 核心指标：
  - 约束满足率
  - 决策置信度
  - 衣物复用率

#### Superpower 2：动态体态档案（Dynamic Body Profile）
- 不是：一次性填写身高体重肩宽。
- 而是：通过多角度照片、穿着反馈、体态变化，持续更新“你对版型的真实偏好”。
- 核心指标：
  - 体态识别误差下降率
  - 用户修正频率下降率
  - 版型适配命中率

#### Superpower 3：衣橱组合缺口计算（Wardrobe ROI Predictor）
- 不是：告诉你“这件像你已有的90%”。
- 而是：计算“这件新衣会新增多少搭配组合、激活多少旧衣、填补哪些场景缺口”。
- 核心指标：
  - 组合增量
  - 旧衣激活率
  - 场景覆盖率

### 1.3 设计原则（v3.0）

| 原则 | 说明 |
|------|------|
| 决策先于美观 | 先满足功能约束，再优化美学呈现 |
| 可解释决策 | 每条推荐必须给出可验证的理由链 |
| 自进化档案 | 体型和衣橱都是动态更新的知识体 |
| 组合价值最大化 | 单品价值由它与衣橱的组合能力决定 |
| 渐进式超能力 | 新用户先获得基础决策，再解锁更高级洞察 |

## 四、模块 B：穿搭决策中枢 -- Style Decision Copilot

### 4.1 功能规格

| 功能 ID | 功能名称 | 优先级 | 功能描述 |
|---------|----------|--------|----------|
| B-01 | 五维输入采集 | P0 | 天气、行程、日期、体态、场景 |
| B-02 | 自动天气获取 | P0 | 温度/湿度/降雨/风力/紫外线 |
| B-03 | 行程选择 | P0 | 通勤/约会/运动/居家/出差 |
| B-04 | 动态体态档案 | P0 | 实时身高/体重/肩宽/腰线/腿型/体态变化 |
| B-05 | 决策推理引擎 | P0 | 约束满足 + 多目标优化 + 可解释决策 |
| B-06 | 决策链输出 | P0 | 每套方案附带决策理由、风险提示、切换选项 |
| B-07 | 一键换一套 | P0 | 基于当前约束快速生成替代方案 |
| B-08 | 穿着记录 | P1 | 记录穿了什么 + 场景 + 舒适度 |
| B-09 | 决策反馈学习；支持连续拒绝/修正反馈，降低同类风格权重，更新体态/偏好档案；记录用户接受率并持续反哺模型 | P1 | 用户行为反哺模型 |
| B-10 | 穿着提醒 | P1 | 定时推送，附带决策摘要 |
| B-11 | 搭配收藏 | P1 | 收藏决策方案而非单品 |

### 4.2 决策推理引擎（三层约束模型）

#### 4.2.1 架构升级：从评分到决策链
```
输入：weather + occasion + date + body_profile + wardrobe_graph
    |
    v
[Layer 1: 硬约束过滤]
  +-- 温度区间 -> 衣物厚薄过滤
  +-- 场合着装要求 -> 正式/休闲过滤
  +-- 活动强度 -> 弹性/透气性过滤
  +-- 身体敏感区 -> 禁忌版型过滤
  +-- 输出：可行域 S_hard
    |
    v
[Layer 2: 多目标优化]
  +-- 穿搭美学评分 / 明日评分
  +-- 衣橱复用率
  +-- 明日可穿性
  +-- 用户偏好匹配
  +-- 输出：候选方案集 S_soft
    |
    v
[Layer 3: 决策解释生成]
  +-- 理由链生成（LLM）
  +-- 风险提示
  +-- 备选方案
  +-- 输出：Top-N 决策方案 + 可解释性报告
```

#### 4.2.2 硬约束规则
- 温度 < 10°C：必须包含外套/长裤/保暖层
- 正式会议：禁用运动鞋/短裤/破洞
- 腰腹敏感：禁用低腰裤/紧身短款
- 雨天：禁用麂皮/浅色麂皮/雪地靴（除非防水款）
- 用户过敏：禁用羊毛/某类金属饰品

#### 4.2.3 多目标优化权重（可配置）
| 目标 | 默认权重 |
|------|----------|
| 身材适配 | 50% |
| 美学评分 | 20% |
| 衣橱复用 | 15% |
| 实用场景 | 10% |
| 用户偏好 | 5% |

### 4.3 决策输出格式（可解释决策卡片）

```
方案一：智能决策（推荐指数 92/100，置信度 85%）
├─ 上装：浅蓝条纹衬衫（修身）
│   ├─ 决策理由：通勤场景首选；肩线设计优化肩腰比；透气性适合 22C
│   └─ 风险提示：浅蓝在阴天对比度略低，可加白色内搭提亮
├─ 下装：深灰高腰直筒西裤
│   ├─ 决策理由：高腰线补偿腿长；直筒版型遮盖腿型；近30天未穿着（复用）
│   └─ 风险提示：久坐可能产生膝盖鼓包，建议站立活动后调整
├─ 鞋：棕色德比鞋
│   └─ 决策理由：比运动鞋正式，比高跟鞋舒适；与裤脚长度匹配
└─ 配饰：银色手表 + 黑色皮带
    └─ 决策理由：冷调肤色适合银色；腰带颜色与鞋呼应

如果下雨（切换成本：60秒换鞋）
├─ 替换鞋：棕色防水切尔西靴
├─ 保留：上装/下装/配饰均无变更
└─ 评分变化：+3（天气适配提升）

不推荐：白色运动鞋
└─ 理由：场合着装冲突（正式会议）+ 雨天不适
```

```
  +-- 行程：用户预设/选择

  +-- 衣橱：图谱查询
```

### 4.5 动态体态档案（Body-Aware Personalization）

#### 4.5.1 多模态体态理解
```

  +-- 视觉推断：正面/侧面/全身照
  +-- 穿着反馈：松紧/舒适度/显胖/显瘦
```

## 五、模块 C：买前防撞参谋 - Wardrobe ROI Predictor

### 5.1 功能规格

| 功能 ID | 功能名称 | 优先级 | 功能描述 |
|---------|----------|--------|----------|
| C-01 | 新衣录入 | P0 | 拍照/截图/链接解析上传新衣 |
| C-02 | 视觉相似度检测 | P0 | CLIP embedding 余弦相似度 |
| C-03 | 属性级对比 | P0 | 颜色/品类/风格/版型多维度对比 |
| C-04 | 动态体态适配评估 | P0 | 新衣版型 vs 用户动态体态档案 |
| C-05 | 组合缺口计算 | P0 | 新衣能新增多少有效搭配组合 |
| C-06 | 旧衣激活分析 | P0 | 新衣能让多少旧衣从“闲置”变“可穿” |
| C-07 | 场景覆盖率提升 | P0 | 填补哪些当前衣橱缺失的场景 |
| C-08 | 购买收益分 | P0 | 综合组合增量、激活率、场景覆盖的ROI评分 |
| C-09 | 性价比评估 | P0 | 基于收益分的购买建议 |
| C-10 | 替代建议 | P1 | 衣橱中是否有功能替代品 |
| C-11 | 愿望清单 | P1 | 收藏待购衣物，跟踪决策状态 |

### 5.2 核心算法：组合缺口计算（Combination Gap Analysis）

#### 5.2.1 算法流程
```
输入：新衣图片/链接
    |
    v
[视觉识别 + 属性提取]
  +-- 品类、颜色、风格、版型、价格
    |
    v
[衣橱相似度检索]
  +-- 向量检索 Top-K 相似衣物
  +-- 属性级加权相似度
  +-- 输出：相似衣物列表 + 相似度分数
    |
    v
[组合增量计算]
  +-- 对衣橱中每一件衣物，计算与新衣的搭配可行性
  +-- 统计：新增有效搭配数
  +-- 统计：激活旧衣数
    |
    v
[场景覆盖率分析]
  +-- 当前衣橱场景覆盖
  +-- 新衣带来的场景增量
    |
    v
[旧衣冲突检测]
  +-- 是否导致某类衣物冗余
  +-- 是否压缩旧衣穿着频率
    |
    v
[Wardrobe ROI Score]
  +-- ROI = w1*组合增量 + w2*旧衣激活率 + w3*场景覆盖提升 - w4*冗余度 - w5*价格/效用比
  +-- 输出：购买建议
```

#### 5.2.2 输出报告
```
买前参谋报告
- 相似衣物：蓝色牛津纺衬衫（相似度 92%）
- 身材适配：修身 -> 适合肩宽，但腰腹区域偏紧
- 组合缺口分析：新衣可新增8套；可激活3件旧衣
- 购买收益分：82/100
- 建议：推荐购买；与卡其色休闲裤、棕色德比鞋同购
- 替代方案：若预算有限，可考虑同品牌H型剪裁款
```

### 5.3 技术实现细节

| 组件 | 实现方式 | 复杂度 |
|------|----------|--------|
| 相似度检测 | CLIP embedding + Qdrant | 中 |
| 属性对比 | 多标签分类头 + 规则匹配 | 低 |
| 组合计算 | 衣橱图谱遍历 + 硬约束过滤 | 高 |
| 场景覆盖 | 场景向量 + 覆盖率计算 | 中 |
| ROI评分 | 加权评分 + 阈值分级 | 低 |

### 5.4 性能优化
- 衣橱规模 < 50：全量组合计算
- 衣橱规模 50-200：图剪枝 + 抽样评估
- 衣橱规模 > 200：近似算法 + 缓存历史结果
- 新衣相似Top-5先过滤，再对候选做组合计算

## 六、模块 D：脸型体型 + 社媒推荐

### 6.1 功能规格

| 功能 ID | 功能名称 | 优先级 | 功能描述 |
|---------|----------|--------|----------|
| D-01 | 脸型录入 | P1 | 用户选择或拍照 AI 分析 |
| D-02 | 动态体态细化 | P1 | 更详细的身体数据 |
| D-03 | 社媒爬取 | P1 | 小红书/微博热门穿搭帖子 |

### 6.2 社媒推荐算法
```
输入：用户脸型 + 体态档案 + 肤色 + 当前穿搭需求
    |
    v
[社媒爬取] -> [特征匹配] -> [风格迁移验证]
    |
    v
排序：综合分 = 可复现度 × 匹配度 × 正向情感分
    |
    v
输出：单品信息 + 推荐理由 + 衣橱匹配度 + 购买链接
```

## 十二、风险矩阵与应对策略

| 风险 | 影响 | 应对策略 |
|------|------|----------|
| 体态推断隐私敏感 | 高 | 端侧推断为主，云侧仅用匿名特征；用户可删除 |
| 组合算法复杂度高 | 中 | 衣橱规模 >100 时启用图剪枝 + 近似算法 |
| 决策可解释性不足 | 中 | 规则优先 + LLM 解释兜底 |
| 记忆补全隐私合规 | 高 | 明确授权 + 数据最小化 + 本地优先 |
| ROI 模型偏差 | 中 | 用户反馈闭环 + A/B 测试校准权重 |
| 冷启动问题 | 高 | 预设 20 件示例衣物 + 快速导入购买记录 |
| 成本/金额控制 | 高 | 控制 AI 调用成本、订阅金额和 CPS 返点，确保整体性价比 |
| AI 推理成本 | 中 | 控制 API 调用与推理预算，优先缓存、蒸馏与批量处理 |
