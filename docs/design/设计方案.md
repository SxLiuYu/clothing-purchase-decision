# 智能衣橱 + AI 搭配师 设计方案

> 痛点：满柜子衣服，每天早上却觉得"没衣服穿"；冲动买了新外套，回家发现没裤子配，只能压箱底。

---

## 一、产品定位

**一句话：** 你的 7×24 私人搭配师 + 智能买前参谋

| 维度 | 说明 |
|---|---|
| 目标用户 | 22-40 岁，关注穿搭但不精通搭配的都市人群 |
| 核心价值 | 少买错、多穿对、告别"没衣服穿" |
| 差异化 | 穿搭推荐（基于身材）+ 买前防撞（相似度检测）+ 脸型体型匹配推荐 |

---

## 二、核心功能模块

### 模块 A：数字衣橱（基础层）

**用户操作：** 给衣服拍照 → 拍照 → 快速录入

| 功能 | 说明 |
|---|---|
| 拍照/选图录入 | 支持单件/批量上传，自动抠图去背景 |
| AI 自动打标 | 自动识别：品类（上衣/裤子/外套/鞋/配饰）、颜色、图案（纯色/条纹/格纹/印花）、材质（棉/羊毛/牛仔等）、风格（休闲/商务/运动/街头） |
| 季节/场合标记 | 用户可补充：春秋/夏/冬，通勤/约会/运动/居家 |
| 手动补标签 | 对于 AI 识别不准的，用户一键修正 |
| 衣橱浏览 | 网格视图 + 分类筛选（按品类/颜色/季节/风格） |

### 模块 B：AI 穿搭推荐（核心层）— 基于身材体型

**触发方式：** 用户输入 / 每日定时推送

**输入维度（五维输入，缺一不可）：**
1. 🌤️ **天气** — 自动获取（温度、湿度、降雨概率、风力）
2. 📍 **行程** — 用户选择或预设（通勤 / 约会 / 运动 / 居家 / 聚会）
3. 📅 **日期偏好** — 今日 / 明日 / 本周摘要
4. 👤 **体型数据** — 身高、体重、肩宽、腰线、腿型、胖瘦分布
5. 🎨 **肤色** — 冷调 / 暖调 / 中性（影响颜色选择）

**输出：** 3 套搭配方案，每套包含：

```
👔 搭配一：通勤商务风（基于你的身材定制）
  ├─ 上装：浅蓝条纹衬衫（微宽松版型，遮住腰腹）
  ├─ 下装：深灰高腰直筒西裤（拉长腿部线条）
  ├─ 外套：藏青色休闲西装（垫肩设计，优化肩腰比）
  ├─ 鞋：棕色德比鞋（圆头显脚小）
  └─ 配饰：银色手表 + 黑色皮带
  身材适配：肩宽腰细 → 垫肩西装强化优势；腿型不直 → 直筒裤修饰
  天气适配：18-24°C，无雨 ✓
  完整度：★★★★☆
```

**搭配逻辑（AI 引擎）— 身材优先：**

**第一层：身材适配评分（权重最高）**
- 扬长避短规则库：
  - 肩宽腰细：垫肩/结构感上衣 + 高腰裤/裙（强化腰线）
  - 腿粗/不直：直筒裤/A字裙/阔腿裤（遮盖腿型）
  - 腰腹有肉：微宽松版型/垂坠感面料（A字裙、H型外套）
  - 腿短：高腰线设计 + 同色系延伸（高腰裤+短款上衣）
  - 脖子短：V领/大领口（拉长颈部线条）
- 单品版型匹配度（修身/宽松/oversize vs 身材特征）

**第二层：穿搭美学评分**
- 颜色和谐度（互补色 / 同色系 / 中性色安全搭配）
- 风格一致性（全休闲 / 全商务 / smart casual）
- 季节适配（薄厚、面料）
- 天气适配（雨天防泼水面料、高温透气面料）

**第三层：实用度评分**
- 完整度（尽量凑齐上+下+鞋+配饰）
- 穿着频率惩罚（过去 30 天内穿过的组合降低优先级）
- 衣橱复用率（优先使用已有单品）

### 模块 C：买前防撞参谋（差异化亮点）

**场景：** 用户在电商平台看中一件衣服，想买但不确定

**功能：**
1. **相似度检测：** 用户上传/输入新衣信息 → AI 扫描衣橱 → "你这件蓝色衬衫和你已有的 1 件相似度 90%，建议搭配灰色西裤一起买"
2. **身材适配预览：** 将新衣与用户体型进行虚拟试穿 → "这件修身款不适合你的腰腹，建议选大一号"
3. **衣橱融合预览：** 将新衣与衣橱中每件单品逐一虚拟搭配，输出哪些能搭、哪些不能搭
4. **性价比评估：** "这件新衣可以和衣橱中 7 件下装搭配，复用率 70%，推荐购买"
5. **替代建议：** "如果你已经有 3 件白色 T 恤，可以考虑其他颜色增加多样性"

### 模块 D：脸型 + 社交媒体推荐（高级层）

**用户画像录入：**
- 脸型（圆脸 / 方脸 / 长脸 / 椭圆 / 心形）— 影响：领口形状、帽子、眼镜
- 体型（瘦 / 标准 / 胖 / 高 / 矮 / 肌肉型）
- 肩宽、腰线位置、腿型
- 肤色（冷调 / 暖调 / 中性）— 影响：颜色选择

**社交媒体爬取 + AI 分析：**
- 在小红书 / Instagram / 微博等平台搜索模特自拍
- 筛选条件：同脸型 + 同体型 + 高好评（点赞/收藏比、正面评论关键词）
- AI 分析图片中的穿搭单品
- 提取关键单品信息（品牌、款式、颜色、购买渠道线索）

**推荐输出：**
```
基于你的脸型（圆脸）+ 体型（微胖）+ 肤色（暖调）

🔥 热门推荐（来自小红书 2.3k 赞）
  单品：V 领垂坠感针织衫（浅驼色）
  推荐理由：V 领拉长脸型，垂坠感显瘦，驼色衬暖调肤色
  类似款跳转 → [淘宝链接] [京东链接]
  
👗 穿搭组合预览
  搭配 1：V 领针织 + 高腰阔腿裤 + 小白鞋
  搭配 2：V 领针织 + A 字裙 + 短靴
```

```

│   - 拍照/上传  │     │  - 用户管理    │     │  - 视觉识别   │
│   - 衣橱浏览   │     │  - 衣橱 CRUD  │     │  - 搭配生成   │
│   - 推荐展示   │     │  - 推荐接口    │     │  - 相似度计算 │
│   - 社交爬取   │     │  - 电商对接    │     │  - 社交分析   │
└──────────────┘     └──────────────┘     └──────────────┘
                                                    │
                              ┌────────────────────┼────────────────────┐
                              ▼                    ▼                    ▼
                       ┌──────────────┐     ┌──────────────┐    ┌──────────────┐
                       │  视觉模型     │     │  天气 API    │    │  电商 API     │
                       │ (CLIP/自训练) │     │ (和风/Open) │    │ (淘宝/京东)   │
```

### 4.2 AI 核心模型

| 模型 | 用途 | 方案 |
|---|---|---|
| **服装视觉识别** | 自动识别品类、颜色、风格、材质 | 基于 CLIP + 微调 / SAM 分割 + 分类头 |
| **相似度计算** | 检测新衣与衣橱的相似度 | CLIP embedding 余弦相似度 + 属性级对比 |
| **身材适配评分** | 根据用户身材推荐合适版型 | 预置穿搭规则库 + LLM 推理 |
| **搭配生成** | 生成穿搭组合 | LLM（Claude/GPT）+ RAG（衣橱知识库）+ 规则引擎 |
| **社交媒体分析** | 爬取 + 分析好评穿搭 | 爬虫（小红书/Ins） + 图像分类 + 情感分析 |
| **脸型体型匹配** | 根据用户特征推荐 | 预置穿搭规则库 + LLM 推理 |

## 五、关键算法设计

### 5.1 穿搭推荐算法（身材优先）

```
输入：天气 + 行程 + 衣橱 + 用户身材数据
  │
  ├─ Step 1: 身材适配筛选（权重 50%）
  │    ├─ 加载用户身材画像（身高/体重/肩宽/腰线/腿型）
  │    ├─ 应用扬长避短规则：
  │    │    ├─ 肩宽腰细 → 优先垫肩/结构感上衣 + 高腰裤
  │    │    ├─ 腰腹有肉 → 优先垂坠感/A字版型/H型外套
  │    │    ├─ 腿粗/不直 → 优先直筒裤/A字裙/阔腿裤
  │    │    ├─ 腿短 → 优先高腰线设计 + 同色系延伸
  │    │    └─ 脖子短 → 优先V领/大领口
  │    └─ 输出：适配身材的单品候选集 S1
  │
  ├─ Step 2: 美学筛选（权重 30%）
  │    ├─ 天气过滤（薄厚、面料）
  │    ├─ 场合过滤（风格匹配）
  │    └─ 季节过滤
  │    └─ 输出：美学候选集 S2 = S1 ∩ 美学条件
  │
  ├─ Step 3: 搭配组合生成（权重 20%）
  │    ├─ 硬规则：上+下+鞋 必须完整
  │    ├─ 颜色和谐度评分（色轮距离）
  │    ├─ 风格一致性评分
  │    ├─ 多样性惩罚（30 天内穿过的组合降权）
  │    └─ 完整度加分（有配饰 +1）
  │    └─ 输出：Top-N 搭配方案
  │
  └─ Step 4: 输出 3 套方案
       ├─ 每套标注：身材适配理由 + 天气适配 + 完整度
       └─ 按总分排序返回
```

### 5.2 相似度检测（买前防撞）

```
输入：新衣图片/描述
  │
  ▼
CLIP Embedding → 向量 V_new
  │
  ▼
与衣橱中每件衣服的 embedding 做余弦相似度
  │
  ├─ 相似度 > 0.9 → "几乎一样，别买了"
  ├─ 相似度 0.7-0.9 → "很相似，建议搭配 XXX 买"
  └─ 相似度 < 0.7 → "风格差异较大，可以考虑"
  │
  ▼
属性级匹配（颜色 + 品类 + 风格）
  │
  ▼
身材适配评估（新衣版型 vs 用户身材）
  │
  ├─ 版型匹配：修身/宽松/oversize 与身材特征对比
  └─ 输出：身材适配度（适合/一般/不适合）+ 理由
  │
  ▼
输出：相似衣物 + 身材适配度 + 搭配建议 + 购买建议
```

### 5.3 社交媒体推荐

```
输入：用户脸型 + 体型 + 肤色
  │
  ▼
爬取社交媒体（小红书/Instagram）热门穿搭帖子
  │
  ├─ 过滤：图片中有清晰人脸 + 穿搭主体突出
  ├─ 图像识别：脸型匹配、体型匹配
  ├─ 情感分析：评论正向率 > 70%
  └─ 排序：点赞数 × 匹配度
  │
  ▼
AI 分析图片中的穿搭单品
  │
  ▼
输出：单品信息 + 推荐理由 + 类似款购买链接
```

## 九、风险与应对

| 风险 | 应对方案 |
|---|---|
| 衣物识别准确率低 | 提供手动修正入口 + 持续迭代模型 |
| 用户录入门槛高 | 批量拍照模式 + 渐进式引导 |
| 身材数据敏感 | 本地加密存储 + 用户可控删除 |
| 社交平台反爬 | 使用官方 API / 降低爬取频率 / 缓存结果 |
| 电商跳转转化低 | 优化推荐精准度 + 提供专属优惠 |
| 冷启动问题 | 提供"穿搭模板库"快速生成首批数据 |
