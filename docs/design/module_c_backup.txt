## 五、模块 C：买前防撞参谋

### 5.1 功能规格

| 功能 ID | 功能名称 | 优先级 | 功能描述 |
|---------|----------|--------|----------|
| C-01 | 新衣录入 | P0 | 拍照或截图上传新衣图片 |
| C-02 | 相似度检测 | P0 | 与衣橱中每件衣物计算相似度 |
| C-03 | 相似度分级 | P0 | >90%：几乎一样；70-90%：很相似；<70%：差异较大 |
| C-04 | 身材适配预览 | P0 | 新衣版型 vs 用户身材特征 |
| C-05 | 衣橱融合预览 | P1 | 新衣与衣橱单品逐一虚拟搭配 |
| C-06 | 搭配数量统计 | P1 | 新衣可与衣橱中几件单品搭配 |
| C-07 | 性价比评估 | P1 | 基于搭配数量的购买建议 |
| C-08 | 替代建议 | P2 | 已有同类衣物过多时的多样性建议 |
| C-09 | 愿望清单 | P1 | 收藏待购衣物，跟踪决策 |

### 5.2 交互流程

```
[用户看到喜欢的衣服]
    |
    +-- 拍照 --> [拍摄新衣]
    |
    +-- 截图 --> [从相册选择]
    |
    v
[AI 识别新衣属性]
    |
    v
[相似度检测]
  +-- 与衣橱中每件衣物做 embedding 余弦相似度
  +-- 属性级对比（颜色/品类/风格）
    |
    v
[身材适配评估]
  +-- 新衣版型 vs 用户身材特征
  +-- 输出：适合/一般/不适合 + 理由
    |
    v
[衣橱融合预览]
  +-- 新衣 + 上装候选 --> 评分
  +-- 新衣 + 下装候选 --> 评分
  +-- 新衣 + 鞋/配饰候选 --> 评分
    |
    v
[输出报告]
  +-- 相似衣物：蓝色衬衫（相似度 90%）x1
  +-- 身材适配：修身款不适合腰腹，建议选大一码
  +-- 搭配预览：可与 7 件下装搭配（复用率 70%）
  +-- 购买建议：推荐购买（搭配数充足 + 身材适配）
```

### 5.3 技术实现方案

| 技术组件 | 选型 | 说明 |
|----------|------|------|
| 新衣识别 | CLIP + 分类头 | 识别新衣品类、颜色、风格 |
| 相似度计算 | CLIP embedding 余弦相似度 | 向量距离 + 属性级加权 |
| 身材适配评估 | 规则引擎 + LLM | 版型 vs 身材特征匹配 |
| 搭配预览 | 衣橱查询 + 规则过滤 | 快速筛选可搭配单品 |

**相似度检测算法：**

```
输入：新衣图片
    |
    v
CLIP Embedding --> V_new (512-d)
    |
    v
与衣橱向量库 Qdrant 检索 Top-K
    |
    +-- 余弦相似度 > 0.9 --> "几乎一样，别买了"
    +-- 余弦相似度 0.7-0.9 --> "很相似，建议搭配 XXX 买"
    +-- 余弦相似度 < 0.7 --> "风格差异较大，可以考虑"
    |
    v
属性级加权相似度
  +-- 颜色权重：30%
  +-- 品类权重：25%
  +-- 风格权重：25%
  +-- 版型权重：20%
    |
    v
综合相似度 = 0.6 x 视觉相似度 + 0.4 x 属性相似度
```
