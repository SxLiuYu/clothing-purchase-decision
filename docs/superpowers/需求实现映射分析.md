## 六、关键差距与建议

### 高优先级

| 差距 | 影响 | 建议 |
|------|------|------|
| 无持久化存储 | 数据重启丢失 | 集成 SQLite（本地）→ PostgreSQL（云端） |
| 无 LLM 可解释生成 | 决策理由链硬编码 | 集成 Claude/GPT 生成 rationale |
| 体态数据无录入接口 | 无法冷启动体态档案 | 新增 `POST /body/profile` 接口 |
| 多目标权重与设计不一致 | 决策逻辑偏离设计 | 权重对齐设计文档的 50/20/15/10/5 |
| 身体敏感区未接入硬约束 | 体态反馈未影响决策 | 在 `outfits.py` 中集成 `feedback_analyzer` |

### 中优先级

| 差距 | 建议 |
|------|------|
| 无 CLIP 相似度计算 | 用 `candidate_score` 模拟，需集成向量检索 |
| 无衣橱搜索 | 基础搜索可基于属性过滤 |
| 无图片上传 | 需集成 MinIO/S3 + 图片处理 |
| 模块 D 社媒推荐 | 后期独立子项目开发 |

### 技术债务

| 问题 | 位置 | 修复方法 |
|------|------|---------|
| `item.dict()` 弃用 | `store.py:28`, `outfits.py:179`, `roi.py:40`, `body.py:10,13` | 改为 `model_dump()` |
| `datetime.utcnow()` 弃用 | `store.py:41,50` | 改为 `datetime.now(datetime.UTC)` |
| `fit_preferences` vs `fit_preference` 字段名不一致 | `store.py:33` vs `feedback_analyzer.py` | 统一字段名 |
| 硬编码的示例候选 | `outfits.py` | 改为从衣橱图谱动态生成 |

## 七、下一阶段建议

根据超级能力方法论，建议按此优先级推进：

### Step 1：修复技术债务（1 天）
- 修复 `dict()` → `model_dump()`、`utcnow()` → `now(UTC)`、`fit_preference` 字段名统一

- 替换 `MemoryStore`
