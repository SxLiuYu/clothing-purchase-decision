### 3.2 超级能力：记忆补全（Auto Closet Cognition）

#### 3.2.1 多源记忆融合
```
[记忆来源]
  +-- 用户主动上传（拍照/选图）
  +-- 电商购买记录导入（订单截图/API授权）
  +-- 社交图片识别（朋友圈/Instagram，隐私授权后）
  +-- 穿着反馈（用户每日确认穿着的衣物）
  +-- OCR识别（快递单/发票/衣物标签）
       |
       v
[记忆融合引擎]
  +-- 去重：同一件衣服多来源合并
  +-- 补全：补全缺失标签
  +-- 置信度：多源验证提高标签置信度
  +-- 冲突解决：以用户修正为准，AI推测为次
       |
       v
[动态衣橱图谱]
  +-- 物品节点（衣物）
  +-- 关系边（可搭配、同色系、同场景、已穿组合）
  +-- 属性节点（颜色、版型、季节、场合）
```

#### 3.2.2 动态标签修正
- 当用户标记某件衣服“太紧”或“显胖”时：
  - 自动修正该衣物的版型标签（修身 -> 标准）
  - 更新体态档案中的敏感区域（腰腹/肩背）
  - 降低该衣物在不适场合的推荐权重
- 形成反馈闭环：穿着反馈 -> 标签修正 -> 推荐优化

### 3.3 衣橱关系图谱
- 不只是列表，而是构建衣物间的关系网络：
  - `可搭配` 边：历史上被用户确认过的搭配
  - `同场景` 边：相同场合高频出现
  - `同色系` 边：颜色空间聚类
  - `替代关系` 边：功能相似但风格不同
- 支持图谱查询：
  - “找出能和这条裤子搭配的所有上衣”
  - “找出最近30天没穿过的通勤外套”
