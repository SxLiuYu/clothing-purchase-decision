### 4.0 能力总览
- 目标：从“穿搭推荐”升级为“穿搭决策代理”
- 核心转变：由“美学排序”转为“约束满足 + 可解释决策”
- 交付物：不只是3套搭配，而是3套“可执行决策方案 + 风险提示 + 切换方案”

### 4.1 场景级需求拆解
| 场景 | 用户任务 | 决策中枢输出 | 成功标准 |
|------|----------|--------------|----------|
| 早晨出门 | 快速决定今日穿搭 | Top-1推荐 + 一键确认 | 10秒内完成决策 |
| 会议/约会 | 场合约束强 | 满足硬约束的稳妥方案 | 0违规项 |
| 突然下雨 | 动态环境变化 | 切换建议 + 换鞋/加外套提示 | 切换成本<1分钟 |
| 长期规划 | 本周穿搭不重样 | 周度复用计划 + 清洗协调 | 30天重复率<20% |

### 4.2 决策链设计
```
输入层
  +- 天气（温度/降雨/风力/紫外线）
  +- 行程（通勤/约会/运动/居家/出差）
  +- 日期（今日/明日/本周）
  +- 动态体态档案
  +- 衣橱关系图谱
       |
       v
硬约束层（不可违反）
  +- 温度约束：厚度/面料/防风/防水
  +- 场合约束：正式度/运动度/暴露度
  +- 体态约束：敏感区域/禁忌版型
  +- 场景约束：久站/久坐/户外时长
  +- 用户约束：过敏材质/必须单品/避免颜色
       |
       v
软优化层（可权衡）
  +- 美学评分：颜色和谐/风格一致
  +- 复用评分：优先低频旧衣
  +- 场景评分：多场合切换能力
  +- 明日评分：考虑洗涤周期
  +- 偏好评分：历史接受模式
       |
       v
决策输出层
  +- 方案1（推荐）：最高综合分
  +- 方案2（稳妥）：约束满足优先
  +- 方案3（个性）：美学偏好优先
  +- 切换预案：下雨/降温/加班替代方案
```

### 4.3 决策卡片规格
```
方案一：智能决策（推荐指数 92/100，置信度 85%）
- 上装：浅蓝条纹衬衫（修身）
  +- 决策理由：通勤场景首选；肩线设计优化肩腰比；透气性适合 22C
  +- 风险提示：浅蓝在阴天对比度略低，可加白色内搭提亮
- 下装：深灰高腰直筒西裤
  +- 决策理由：高腰线补偿腿长；直筒版型遮盖腿型；近30天未穿着（复用）
  +- 风险提示：久坐可能产生膝盖鼓包，建议站立活动后调整
- 鞋：棕色德比鞋
  +- 决策理由：比运动鞋正式，比高跟鞋舒适；与裤脚长度匹配
- 配饰：银色手表 + 黑色皮带
  +- 决策理由：冷调肤色适合银色；腰带颜色与鞋呼应

如果下雨（切换成本：60秒换鞋）
- 替换鞋：棕色防水切尔西靴
- 保留：上装/下装/配饰均无变更
- 评分变化：+3（天气适配提升）

不推荐：白色运动鞋
- 理由：场合着装冲突（正式会议）+ 雨天不适
```

### 4.4 多目标优化权重（可配置）
| 目标 | 默认权重 | 说明 | 可配置项 |
|------|----------|------|----------|
| 身材适配 | 50% | 硬性满足 + 版型优化 | 体态敏感区权重 |
| 美学评分 | 20% | 颜色和谐、风格一致 | 色轮距离阈值 |
| 衣橱复用 | 15% | 激活低频旧衣 | 30天/90天未穿阈值 |
| 实用场景 | 10% | 多场合切换能力 | 场景定义 |
| 用户偏好 | 5% | 历史接受模式 | 学习速率 |

### 4.6 决策反馈学习
- 用户接受率 -> 调整该场景下美学权重
- 用户手动替换单品 -> 学习单品替换偏好
- 用户标记“不舒服” -> 触发体态档案更新
- 连续拒绝某类搭配 -> 降低该类风格权重

### 4.7 动态体态档案（Body-Aware Personalization）

#### 4.7.1 多模态体态理解
```
[体态数据来源]
  +-- 用户输入：身高/体重/肩宽/腰线/腿型（基础数据）
  +-- 视觉推断：正面/侧面/全身照（可穿戴设备可选）
  +-- 穿着反馈：松紧/舒适度/显胖/显瘦（隐式标注）
  +-- 体态变化：周期性体重、围度变化趋势
       |
       v
[体态特征编码]
  +-- 静态特征：身高、肩宽、腿长、腰线位置
  +-- 动态特征：坐姿驼背、骨盆前倾、高低肩
  +-- 敏感区域：腰腹、肩背、小腿（用户高频反馈区域）
  +-- 审美编码：显肩宽偏好、遮腰腹偏好、拉长腿部偏好
       |
       v
[自进化档案]
  +-- 每次穿着反馈自动更新敏感权重
  +-- 当用户多次修改AI推荐的“修身”为“宽松”时：
       - 自动下调该衣物的修身标签
       - 上调用户对版型的宽松偏好权重
```

#### 4.7.2 体态推断示例
- 用户上传侧面照 -> 检测到圆肩/探颈 -> 自动推荐落肩款权重降低，挺括版型权重上升
- 用户反馈“这件衬衫显肚子” -> 自动强化腰腹遮挡规则，建议选大一码或H型
